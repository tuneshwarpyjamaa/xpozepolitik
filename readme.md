Website: Karnataka MP TrackerVersion: 1.0Date: November 4, 2025Product Goal: To provide citizens with an easily accessible, accurate, and concise repository of information about their current Members of Parliament (MPs) representing Karnataka in both Lok Sabha and Rajya Sabha.1. Goals and ObjectivesPrimary Goal: Enhance political transparency and citizen engagement by creating a single, reliable source for current Karnataka MP data.Key Objectives:Achieve a high level of data accuracy for all listed MPs.Ensure the website is intuitive and easy to navigate for users of all technical skill levels.Provide concise, scannable profiles that clearly communicate an MP's background, work, and performance.Maintain mobile-first responsiveness to ensure accessibility on all devices.2. Target AudienceGeneral Public/Voters: Seeking to know their representative, their background, and their performance.Students/Researchers: Needing reliable, consolidated data for academic or research purposes.Journalists/Media: Requiring quick, verified facts and profiles.Political Activists/NGOs: Monitoring public representatives' activities.3. Functional Requirements (What the product must do)3.1 MP Listing Page (Homepage/Primary View)IDRequirementDescriptionFR 3.1.1MP Directory DisplayDisplay a clear, sortable, and filterable list of all current Lok Sabha (28) and Rajya Sabha (12) MPs from Karnataka.FR 3.1.2Core MP InformationEach entry must concisely display the MP's Name, Constituency (for Lok Sabha) / State Represented (for Rajya Sabha), Political Party, and a professional photograph.FR 3.1.3Filtering & SortingUsers must be able to filter the list by: Lok Sabha/Rajya Sabha, Political Party, and Region/District. Users must be able to sort by Name (A-Z) and Constituency Name.FR 3.1.4Search FunctionalityA prominent search bar must allow users to search the list by MP Name, Constituency Name, and Political Party.3.2 Detailed MP Profile PageIDRequirementDescriptionFR 3.2.1Basic Bio DataDisplay the MP's Full Name, Constituency/State, Party, Date of Birth, Place of Birth, and Education Details (Highest Qualification).FR 3.2.2Professional/Political Journey (Concise)A brief, scannable section listing key Previous Posts Held (e.g., MLA, Minister, previous MP terms) and their Profession before/during politics.FR 3.2.3Contact InformationDisplay official Parliamentary/Constituency Office Address, Official Email ID, and Official Contact Number.FR 3.2.4Social Media & Web LinksInclude links to official Social Media Profiles (Twitter/X, Facebook) and the MP's official Digital Sansad/Parliament Profile.FR 3.2.5Performance SummaryDisplay a concise summary of key performance indicators (KPIs). This data should be prominently displayed and ideally updated periodically (e.g., quarterly or annually).FR 3.2.5 - Detailed Performance Summary (Concise Data Points)Attendance: Percentage in Lok Sabha/Rajya Sabha sessions (compared to National/State average).Questions Raised: Total number of questions asked in Parliament (compared to National/State average).Debates Participated: Total number of debates participated in (compared to National/State average).MPLADS Utilisation: Funds released and utilised under the MPLADS scheme (or a simple 'Utilisation Status').4. Non-Functional Requirements (How the product must perform)IDRequirementDescriptionNFR 4.1AccuracyAll MP data must be sourced from official government/parliamentary records (e.g., Digital Sansad, Election Commission). A Data Source link must be provided on the profile page.NFR 4.2PerformanceThe website must load in under 3 seconds on standard broadband connections and mobile networks.NFR 4.3SecurityStandard website security (HTTPS) is mandatory. As no user data is collected, high-level user data security is not required, but site integrity must be protected.NFR 4.4Design/UXThe design must be clean, modern, and non-partisan in colour scheme and layout. The primary font size must be accessible for general reading.NFR 4.5ResponsivenessThe site must be fully responsive and mobile-friendly, adapting seamlessly to screen sizes from mobile phones to desktop monitors.5. Technical RequirementsIDRequirementDescriptionTR 5.1PlatformRecommended to use a modern, secure CMS or framework (e.g., WordPress, React, Django) for ease of maintenance and scalability.TR 5.2Data UpdateA simple Admin Panel is required to allow non-technical staff to update MP data, especially in the event of new elections, by-polls, or term changes.TR 5.3HostingRequires reliable, scalable web hosting to handle potential traffic spikes, especially around election times.6. Success MetricsTraffic: Average monthly unique visitors (Target: 5,000+ within 6 months).Engagement: Average time spent on an MP Profile Page (Target: > 1 minute).Accuracy: Zero reported data errors within the first 3 months post-launch.Search/Filter Usage: Percentage of users who use the search or filter function (Target: > 50% of users).
